Nori layered architecture

从用户意图到可审计内容产物

这张图把当前 Nori 代码架构按业务流重新铺开：User Interaction 创建会话和任务； Information Layer 把输入、市场证据、资产和策略编译成上下文；Agent Layer 执行业务能力； Memory Layer 负责跨 session 的长期事实沉淀；Workflow Runtime 把执行过程、Human Gate 和 artifact trace 串起来。

interaction3入口类型：future UI、API/CLI、live scripts。
runtime4sessions、context、memory、workflows 是一级系统目录。
agents5画像、市场、规划、生成、学习闭环。
prooftraceWorkflowRun、ContextBundle、artifact refs、review gates。
