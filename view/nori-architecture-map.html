Nori architecture map

从用户意图到内容闭环

Nori 现在的核心不是一堆散落 Agent，而是一条有契约的运营内容流水线： 先冻结用户意图，再规划账号运营，再路由生成文本或图片，最后用审核和指标回流驱动下一轮策略。

5业务域：用户画像、市场分析、上下文建设、内容生成、学习闭环。
11stage package：每个复杂 stage 拥有自己的 entry 和 package contract。
0stage-local schema / prompts / inputs / refs / builder 残留文件。

业务闭环

业务上 Nori 是账号代运营的内容生产系统，不是单次文案生成器。每个阶段都产出可复用的结构化结果。

生成链路

生成层采用“总入口路由 + 专门子 Agent”的结构。总入口不吞掉领域能力，只决定走文本、图片还是完整内容包。

IntentContract

冻结目标、必含词、语气、交付物和禁忌，避免后续阶段重新猜用户意图。

ArtifactStore

长流程可把 stage JSON 写成 checkpoint，并通过 manifest 恢复。

LLMFactory

所有 Agent 通过注入网关调用 LLM，不直接依赖底层 llms 模块。

GenerationAgent

artifact_type 是粗粒度路由开关。它让产品上有统一入口，同时保留专门 Agent 的边界。

完整小红书内容包note_package
只生成正文草稿text
只生成封面图片image

ContentProducerAgent

把 ContentTask、Skill、Assets 装配成 ContentPackage，保存 note / cover 产物快照。

NoteMakerAgent

负责 skill 选择、素材整理、标题正文标签生成，不读取原始图片 bytes。

CoverDirectorAgent

负责参考图选择、封面 prompt、生图调用和输出持久化。

entry

*_agent.py 或 stage 主文件只负责编排和依赖注入。

package contract

package.py 聚合同一 Agent 的类式输入、prompt、引用或装配逻辑。

policy / normalizer

纯规则、fallback、LLM 输出清洗继续拆开，避免 package 变成大杂烩。

core

nori.core 放跨 stage 契约、LLM 网关、artifact/checkpoint 和 base class。
