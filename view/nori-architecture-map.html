Nori current architecture

现在的 Nori：意图、运行时、Agent 与产物闭环

这张图反映当前代码状态：没有独立 Domains 层，业务能力已经内化到五个 agent capability； session、context、memory、workflow 是一级运行时目录；LangGraph 只承载粗粒度业务节点和 Human Gate，细碎实现留在 agent package 内。

端到端业务流

用户不是直接调用某个生成器；系统先建立 session/task/context，再由 workflow 执行业务 agent，并把每个阶段产物记录为 artifact。

五个 Agent Capability

现在合理的边界不是 Domains，而是 agent 内部能力所有权。每组 agent 对外提供 facade，对内保留 stage package、policy、normalizer、state 等模块。

User Profiling

nori.agents.user_profiling

  • IntakeAgent 理解文本与图片资产。
  • AccountPlannerAgent 生成账号定位。
  • 拥有 user/account/brand profile 模型。

Market Analysis

nori.agents.market_analysis

  • 接入 XHS 数据采集结果。
  • 抽取 NoteSkill 和 session report。
  • 把外部证据转成生成可用能力。

Planning

nori.agents.planning

  • ClientBrief -> AccountOperationProject。
  • 生成 KPIPlan、ContentCalendar、ContentTask。
  • 任务规划与 context 编译解耦。

Content Generation

nori.agents.content_generation

  • ContentSpecAgent 冻结生成策略。
  • ArtifactGenerationAgent 执行 spec。
  • NoteMaker/CoverDirector 是专业生成器。

Learning Loop

nori.agents.learning_loop

  • ReviewGate 检查合规与一致性。
  • MetricsSnapshot 记录后验数据。
  • StrategyIteration 产生下一轮策略信号。

LangGraph 执行图

产品图不应该把每个 helper 都画成 node。LangGraph 只承载需要 checkpoint、retry、Human Gate 或跨 agent 编排的粗粒度节点。

Workflow Runtime

Graph 只管粗粒度业务节点、Human Gate、状态传递和 artifact trace；业务判断仍留在 agent package 内。

一级目录职责

这部分是现在最重要的架构对齐：memory 和 context 是一级目录，agent 不再拥有全局状态；agent 只消费 runtime 给它的输入。
