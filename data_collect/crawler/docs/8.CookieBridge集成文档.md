# 8. CookieBridge 集成文档

## 1. 简介

CookieBridge 是 MediaCrawlerPro 的浏览器 Cookie 自动提取与同步桥接服务，由 Chrome Extension 和 Python Tornado Server 两部分组成。它可以替代传统的 Excel/MySQL 手动管理 Cookie 的方式，实现 Cookie 的零手动自动同步。

### 架构概览

```
Chrome Extension (MV3)  ── WebSocket ──>  CookieBridge Server (Python/Tornado)
                                                     ↓ HTTP REST API
                                          MediaCrawlerPro-Python 爬虫
```

1. 用户在 Chrome 浏览器中登录各平台
2. Chrome Extension 自动提取 Cookie 并通过 WebSocket 推送到 CookieBridge Server
3. MediaCrawlerPro-Python 爬虫启动时通过 HTTP API 从 CookieBridge Server 获取 Cookie

## 2. 三种 Cookie 来源对比

| | Excel (`xlsx`) | MySQL (`mysql`) | CookieBridge (`cookie_bridge`) |
|---|---|---|---|
| Cookie 获取 | 手动复制粘贴到 Excel | 手动写入数据库 | 浏览器登录后自动同步 |
| Cookie 更新 | 过期后需手动重新提取 | 过期后需手动更新 | 浏览器在线时自动保持最新 |
| 多账号管理 | Excel 不同行 | 数据库不同记录 | 不同 Chrome Profile 自动区分 |
| 登录状态感知 | 无 | 无 | 实时检测 |
| 状态持久化 | 不支持 | 支持（标记 status=-1） | 不持久化（仅内存标记） |
| 适用场景 | 快速测试、少量账号 | 生产环境、需要状态追踪 | 自动化运维、多账号频繁更新 |

## 3. 配置方法

### 3.1 前置条件

1. 已部署并启动 CookieBridge Server（默认端口 8274）
2. 至少一个 Chrome Profile 已安装 CookieBridge Extension 并连接到 Server
3. 在 Chrome 浏览器中登录了目标平台

### 3.2 配置步骤

**方式一：修改配置文件**

编辑 `config/base_config.py`：

```python
# 将账号池类型改为 cookie_bridge
ACCOUNT_POOL_SAVE_TYPE = "cookie_bridge"

# CookieBridge 服务地址（默认 http://localhost:8274）
COOKIE_BRIDGE_URL = "http://localhost:8274"
```

**方式二：通过环境变量**

```bash
export ACCOUNT_POOL_SAVE_TYPE=cookie_bridge
export COOKIE_BRIDGE_URL=http://localhost:8274  # 可选，默认值

uv run main.py --platform xhs --type search
```

### 3.3 验证连接

```bash
# 检查 CookieBridge Server 是否在线
curl http://localhost:8274/ping

# 查看已注册的账号列表
curl http://localhost:8274/api/accounts

# 查看特定平台的 Cookie
curl http://localhost:8274/api/cookies/xhs
```

## 4. 工作原理

### 4.1 Cookie 加载流程

```
爬虫启动
  → AccountPoolManager.async_initialize()
  → load_accounts_from_cookie_bridge()
    → GET /api/accounts  (获取所有已注册的客户端列表)
    → 遍历每个客户端:
       → 检查该客户端在目标平台是否有 cookie (platforms[platform].has_cookies)
       → GET /api/cookies/{platform}?client_id={client_id}  (获取具体 cookie)
       → 封装为 AccountInfoModel 加入内存账号池
    → 爬虫使用内存中的账号池进行请求
```

### 4.2 账号标识

- `account_name`：优先使用 CookieBridge 返回的平台昵称（`nicknames[platform]`），如无则使用 `client_id`
- `id`：内存中自增序号（CookieBridge 不使用数据库 ID）

### 4.3 状态管理

CookieBridge 模式下，账号失效**仅在内存中标记**，不会回写到 CookieBridge Server。原因：
- CookieBridge 的 Cookie 由浏览器端自动维护和刷新
- 下次爬虫重启时会重新从 CookieBridge 拉取最新 Cookie
- 如果某个 Cookie 确实失效了，用户在浏览器中重新登录即可

## 5. 多账号使用

1. 在 Chrome 中创建多个 Profile（`Chrome 右上角头像 → 添加个人资料`）
2. 每个 Profile 安装 CookieBridge Extension
3. 在不同 Profile 中登录同一平台的不同账号
4. 每个 Profile 的 Extension 会自动分配独立的 `client_id`
5. 爬虫启动时会自动加载所有可用账号到账号池

## 6. 注意事项

- CookieBridge Server 必须在爬虫启动**之前**运行，否则会报连接错误
- 如果 CookieBridge Server 不可用，会抛出异常并提示检查服务状态
- CookieBridge 模式与 Excel/MySQL 模式互斥，同一时间只能使用一种
- 代理 IP 功能与 CookieBridge 模式完全兼容，可同时使用
